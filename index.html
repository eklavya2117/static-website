About This Project

This project demonstrates a complete DevOps workflow for deploying a static website. The application is built using pure HTML and CSS, packaged into a Docker image, and deployed automatically to an EC2 server using Jenkins CI/CD. The goal is to showcase clean automation, reproducibility, and production-ready deployment practices.

Key Features

Fully Static

No backend dependencies, resulting in fast load times and high security.

Responsive Design

Optimized layouts for mobile, tablet, and desktop screens.

Dockerized

Packaged inside an Apache Docker container for consistent deployments.

CI/CD Enabled

Automated build, image creation, and deployment using Jenkins.

Application Architecture

  • User accesses the website via browser.
  • Request hits Apache running inside a Docker container.
  • Docker container runs on an EC2 instance.
  • Docker image is built and pushed via Jenkins pipeline.
  • Artifacts are versioned and archived in Jenkins.

CI/CD Pipeline Flow

  1. Source code checkout from GitHub.
  2. Build stage creates optimized static artifacts.
  3. Artifacts are archived for traceability.
  4. Docker image is built using build output.
  5. Image is pushed to Docker Hub.
  6. Latest image is deployed on EC2 automatically.
